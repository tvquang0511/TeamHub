# TeamHub — C4 Model (Level 1–2)

> Mục tiêu: mô tả hệ thống theo chuẩn C4 đến Level 2 (System Context + Container).
> Diagrams dùng PlantUML (không phụ thuộc thư viện C4-PlantUML).

---

## Level 1 — System Context

```plantuml
@startuml
skinparam linetype ortho
left to right direction

actor "User" as User
rectangle "TeamHub\n(Kanban + Chat + Reminders + Analytics)" as TeamHub

cloud "SMTP Provider" as SMTP
cloud "Object Storage\n(MinIO / S3)" as S3

User --> TeamHub : Uses via browser\n(HTTPS + WebSocket)
TeamHub --> SMTP : Sends reminder emails
TeamHub --> S3 : Stores files\n(avatars/backgrounds/attachments)

@enduml
```

---

## Level 2 — Containers

```plantuml
@startuml
skinparam linetype ortho
left to right direction

actor "User" as User

node "Client" as Client {
  component "Browser" as Browser
}

node "Edge" as Edge {
  component "Nginx\n(reverse proxy, prod-like)" as Nginx
}

node "TeamHub" as TeamHub {
  component "Frontend\n(React + Vite)" as FE
  component "Backend API\n(Express + TS)" as API
  component "Socket.IO\nGateway" as WS
  component "Worker\n(BullMQ consumers)" as Worker
}

database "PostgreSQL" as PG
node "Redis" as Redis
node "Object Storage\n(MinIO / S3)" as S3
cloud "SMTP Provider" as SMTP

User --> Browser : Interacts
Browser --> Nginx : HTTPS
Nginx --> FE : serve static
Nginx --> API : proxy /api
Nginx --> WS : proxy /socket.io

Browser --> API : REST (JWT)
Browser --> WS : WebSocket (JWT)

API --> PG : Prisma (read/write)
Worker --> PG : read/write\n(reminders/metrics)

API --> Redis : BullMQ enqueue\n+ rate limit\n+ cache (optional)
Worker --> Redis : BullMQ consume

API --> S3 : presign PUT/GET
Browser --> S3 : upload/download\n(presigned URLs)

Worker --> SMTP : send emails

note right of Redis
Redis is required for:
- BullMQ queues (reminders/emails/analytics)
- Cache-aside (optional)
- Rate limiting (optional)
end note

@enduml
```
